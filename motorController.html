Designing a Motor Controller

As part of UTSM's Prototype Powertrain team, I'm currently implementing the motor controller unit for the Prototype car which will compete in the 2024 Shell Eco-Marathon Americas Competition. The design for this car's motor controller is entirely new because it will fit in the team's first fully electric car. Due to competition regulations, the motor control unit must be fully self-designed.

Working alongside Andrew Radke and Maya Edie-Maxsom in the Prototype Powertrain team.

Layout planning

In the inital planning stages, we used a previous iteration of the motor controller PCB to create a model for how our motor controller would work. The choice to use a PCB was made on the basis that the system needed to be small to fit inside a one-passenger racecar, and because it would be able to supply high power. Using a combination of encoders, drivers, an Arduino microcontroller, and other components, we started our motor controller by designing the PCB layout using Altium Designer.

Future Design Steps

With the PCB designs nearly complete, the team will order the completed PCBs in the next few weeks. In the meantime, I will implement the C code that is need to control the decoders and drivers via the Arduino microcontroller. After the PCBs have arrived, the priority is to ensure that our current design works with the rest of the electrical system and diagnose any weaknesses or issues with the rest of the Prototype team.
